# LANE M2 — Production image + deploy profile
[wave 1 · depends: M1 MERGED · taxonomy INFRA · model Sonnet 5 · P6.5: no]

## 1. Objective
Ship the runnable miman stack: hardened image built from the LOCAL patched tree, and the
`deploy/` profile (compose: miman + qdrant + appdb; env contract; deploy.sh with behavior
smokes). Realizes spec 03-M §4.3-4.4, §5.4-5.5, §6.1, §9.3, §12 items 4-7.

## 2. Scope / Non-scope
In: root `Dockerfile`, `.dockerignore`, `docker/entrypoint.sh` (alembic upgrade head →
exec uvicorn), `deploy/{docker-compose.yml,.env.example,deploy.sh,README.md}`, root
`Makefile` additions (`miman-test|docker-build|deploy-check|release-all|compose-config` —
prefixed; never modify upstream targets), `scripts/deploy-check.sh`,
`scripts/release-all.sh` (tag prefix `miman-v`). Out: workflows (M3), server code (done in
M1 — if a server change seems needed, PARK). Do-not-touch: `server/Dockerfile`,
`server/docker-compose.yaml` (upstream dev stack stays intact), `server/requirements.txt`.

## 6. Requirement checklist
R1 image from local tree, no PyPI mem0ai layer (spec §6.1; proof = acceptance A3).
R2 non-root 10001, /app/history writable, EXPOSE 8000, HEALTHCHECK /healthz, no --reload.
R3 entrypoint runs `alembic upgrade head` idempotently before uvicorn.
R4 compose per spec §5.4: pinned `postgres:18-alpine` + `qdrant/qdrant:v1.18.3-unprivileged`
   (image-version policy — re-check latest stable at implementation; record actual pins),
   127.0.0.1 binds only, healthchecks, no-new-privileges, private network, no dashboard.
R5 `.env.example` = spec §9.3 exactly (MEM0_VECTOR_STORE=qdrant, MEM0_TELEMETRY=false,
   MIMAN_ENV, `_v<N>` collection).
R6 deploy.sh: P6 flow + smokes — add/search/delete round-trip AND provisioned member key
   403 on `POST /reset` + unscoped `GET /memories` (uses M1's provisioning script via
   docker exec).
R7 deploy-check.sh: P9 checks + provider==qdrant assert + `_v<N>` + telemetry-false +
   no `:latest` + no 0.0.0.0 + no tracked .env + `miman-v` prefix in release-all.
R8 release-all.sh: P10 guards, tag `miman-v${VERSION}`, creates GitHub Release (routes the
   router) — but NEVER run it in this lane.

## 7. Call-graph / data-flow map
Boot chain only (spec §10.1): compose → entrypoint → alembic → uvicorn → /healthz. Verify
by running, not tracing.

## 8. Implementation plan
(1) .dockerignore → (2) Dockerfile + entrypoint, build green → (3) compose + env → (4)
deploy-check.sh → (5) deploy.sh + smokes against a live local stack (stub/real embedder
per available key; record which) → (6) Makefile targets → (7) release-all.sh (bash -n
only) → (8) LETA_PATCH.md rows + tracker.

## 9. Acceptance criteria
A1 `make miman-docker-build VERSION=0.0.0-audit` green.
A2 `bash scripts/deploy-check.sh` green.
A3 image provenance: `docker run --rm --entrypoint python <img> -c "import mem0,
   subprocess; print(mem0.__file__)"` shows /app tree AND
   `docker history <img> | grep -c "pip install mem0ai"` == 0 (no floating install).
A4 `cd deploy && cp .env.example .env` (+ filled secrets) → `bash deploy.sh` completes:
   /healthz, /readyz, add→search→delete, member-key 403 pair — paste output.
A5 container user != root (`docker run --rm --entrypoint id <img>`).
A6 §12 files-changed match.

## 10. Test cases

test). deploy-check self-test: run twice, idempotent.

## 11. Validation commands

`bash -n scripts/*.sh deploy/deploy.sh docker/entrypoint.sh`.

## 13. Risks, blast radius, rollback
Blast radius: nothing running in prod yet; local only. Risks: alembic cwd/ini mismatch
(inspect first); requirements filter regression silently reinstalling PyPI mem0ai (A3 is
the tripwire); stale image pins (re-verify latest stable at implementation per policy).
Rollback: revert PR; images are local until MR1.

## 14. Report format
Template terse report. Record actual image pins chosen + evidence they were latest stable
at implementation time. deploy/README.md must state the rollback path (deprecated stack)
per spec §12 item 12.
