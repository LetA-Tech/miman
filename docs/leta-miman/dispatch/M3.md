## 1. Objective
Wire miman into the repo's CI/CD conventions without breaking upstream pipelines:
`miman-checks.yml` (PR gate), `miman-cd.yml` (DOCR publish, dispatched by the release
router on `miman-v*`), `miman-upstream-sync.yml` (weekly drift ‚Üí sync PR). Realizes spec
03-M ¬ß4.5, ¬ß6.2-6.4, ¬ß12 items 8-9.

## 2. Scope / Non-scope
In: three new workflow files; ONE additive arm in `release.yml` case block; ci-gate
registration (filter + call job + needs). Out: cutting any release (MR1); modifying any
upstream package workflow beyond the router arm; secrets creation
(`DIGITALOCEAN_ACCESS_TOKEN` = Lucas gate ‚Äî verify existence via
`gh secret list`, PARK the CD *verification* (not the file) if absent and record).
Do-not-touch: `cd.yml`, all `*-cd.yml`/`*-ci.yml`/`*-checks.yml` of other packages.

## 6. Requirement checklist
R1 `miman-checks.yml`: `workflow_call` + `workflow_dispatch`; jobs = ruff (LetA paths) ‚Üí
   pytest (LetA suite) ‚Üí `bash scripts/deploy-check.sh` ‚Üí docker build (no push).
R2 ci-gate registration per its own header convention: `miman:` path filter (spec ¬ß6.2
   list), call job, entry in gate `needs`.
R3 `miman-cd.yml`: `workflow_dispatch` inputs tag/prerelease; guard
   `startsWith(inputs.tag, 'miman-v')`; validate job (tag-on-main ancestor, lint/test/
   deploy-check) ‚Üí publish job (doctl DOCR login, buildx, push
   `miman:<tag>` + `miman:<sha>`, OCI labels, GHA cache) ‚Äî port P11 steps.
R4 Router arm: `miman-v*)  workflow="miman-cd.yml" ;;` inserted ABOVE `v*)` with a
   comment; diff to release.yml = exactly that hunk.
R5 `miman-upstream-sync.yml`: weekly cron + dispatch; spec ¬ß6.4 behavior (newest upstream
   `v*` tag vs LETA_PATCH.md base; no-drift green; clean-merge ‚Üí branch + PR with
   protected-file checklist in body; conflict ‚Üí fail loudly; NEVER auto-merge).
R6 No workflow may publish to PyPI/npm (grep-proof).

## 8. Implementation plan
(1) miman-checks.yml + gate registration ‚Üí (2) push branch, confirm gate triggers and is
green on the PR itself (self-verifying) ‚Üí (3) miman-cd.yml + router arm ‚Üí (4) sync
workflow ‚Üí (5) dry-runs: `gh workflow run miman-cd.yml -f tag=miman-v0.0.0-dry` expecting
validate-fail-fast on nonexistent tag (proves wiring), `gh workflow run
miman-upstream-sync.yml` expecting "no drift" green ‚Üí (6) LETA_PATCH.md + tracker.

revert PR; workflows are inert without tags.
